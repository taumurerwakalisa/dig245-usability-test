A Usability Test for Goodreads.com

What is a usability test?

A usability test is a method that is used to check how user-friendly and easy to use a website is.

This test invloves giving a user 5 realistic tasks to complete on the goodreads website while observing and making comments on their interactions.

In this test, we also note down how long it took the user to accomplish each task.

What is Goodreads.com?

Goodreads is a website owned by Amazon.com, that allows users to get book recommendations, rate and review books, and engage with a community of readers.

TASK 1: MAKE AN ACCOUNT/SIGN IN

Our first task was for the user to go to the website and sign in or make an account (for first time users).

When the user got to the website, they were quickly able to find the 'Log in' section. Even better, they were able to link goodreads to their Amazon account in order to sign in.

Overall, this was an easy task for our user.


Time: 5s

TASK 2: FIND YOUR FAVORITE BOOK

The second task was for the user to search for a book of their preference.

This task was fairly easy because the search bar was not difficult to spot. The book immediately popped up when the user hit search.


Time: 11s

TASK 3: ADD A BOOK TO YOUR "WANT TO READ" LIST

After finding their favorite book, the user had to add it to a 'Want to read' list for future reference.

When prompted to do so, the user immediately spotted the green 'Want to read' button next to the book and added the book to the list.

Overall, this was a simple task because the button was not hard to spot.


Time: 3s

TASK 4: JOIN READING GROUP OF INTEREST

The next task was for the user to find and join a reading group they found interesting. A reading group is where readers can discuss books and share recommendations.

The user easily navigated to the community tab, where they found the groups' section. However, the user faced some difficulties when trying to join one:

  • They noticed that some groups were invite-only and there were no identifiers for which ones were or weren't.
  • The user had to add birthday information to join the group.

One recommendation for improvement would be to separate the groups into invite-only and all-access groups.


Time: 1min 45s

TASK 5: LIKE SOMEONE'S POST

This task required the user to like a post made by another goodreads user.

When looking for posts to like, the user was not sure of which ones were likeable i.e. book reviews, profile posts, or comments?

Another issue arose when the user could not find a like button, but eventually did, after some time perusing the page.

After all the trouble with finding a like button, the user made a comment about how the page looked cluttered, indicating an unpleasant design.


Time: 2min 19s

System Usability Scale (SUS)

The System Usability Scale is a measure of a system's usability by prompting users to rate their experience after using a product, on a scale of 1(Strongly Disagree)-5(Strongly Agree), for ten statements.

Conclusion

Goodreads had some issues affecting its ease of use;

  • First, it is difficult to identify the option to like someone’s post.
  • Finding a reading group is easy. However, there are still more steps to complete before actually joining one e.g. adding your birthday info.
  • A user is not able to distinguish which groups are invite-only or all-access.

Generally, the website can be navigated easily, even though there are some minor issues that need to be improved like the ones mentioned above and the way items are arranged to avoid clutter on some of the pages.
